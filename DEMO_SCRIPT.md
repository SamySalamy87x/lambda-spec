# λ-SPEC demo script — 2 minutes 35 seconds

## 0:00–0:15 — Problem

**Voice-over**

“Most dashboards inspect a system internally. They tell us how it performs now, but not whether it only looks healthy because the environment is holding it up. λ-SPEC measures that hidden dependency.”

**Screen**

Show the landing statement and formula. Do not explain the full mathematics.

## 0:15–0:35 — Load evidence

**Voice-over**

“I can upload a CSV, but I will use the network-support demonstration. I map service output, inverse latency, and quality as the system; demand cycle and backup support as the environment.”

**Screen**

Show the loaded dataset, column roles, method, and rolling window. Press **Run resilience analysis**.

## 0:35–1:05 — Global result and trajectory

**Voice-over**

“The product returns λ, the measured coupling; Φ, its complementary fragility; and dλ/dt, the trajectory. The global score alone is only a snapshot. Here the rolling line is weakening, exposing progressive decoupling before complete failure.”

**Screen**

Point to the four metrics, then the rolling λ/Φ chart. Emphasize the negative slope.

## 1:05–1:30 — Withdrawal stress test

**Voice-over**

“Next, λ-SPEC progressively replaces environmental signal with independently permuted observations. This is not a literal forecast. It is a controlled stress envelope showing how fast measured coupling disappears when environmental information is removed.”

**Screen**

Show the environmental-withdrawal chart and the threshold line.

## 1:30–1:52 — Driver ranking and evidence boundaries

**Voice-over**

“The driver ranking removes each environmental variable and measures the loss of λ. Every conclusion is separated into evidence and limits. The system explicitly states that coupling is association, not proof of causation.”

**Screen**

Show the driver bars, evidence list, and limitations.

## 1:52–2:15 — GPT evidence interpreter

**Voice-over**

“The optional GPT interpreter receives only the metric snapshot and is instructed not to invent variables, diagnose, certify, or guarantee outcomes. Without an API key, the product still returns a deterministic local summary, so the workflow never fails.”

**Screen**

Press **Interpret the evidence** and show the engine label. Use the question: “What should I investigate next?”

## 2:15–2:35 — Product completeness

**Voice-over**

“The same application works entirely in the browser, keeps uploaded data local, exports JSON and Markdown, prints a PDF report, runs in Docker, and includes automated Python and JavaScript tests. λ-SPEC does not build another chatbot. It builds a measurable instrument for detecting lost support.”

**Screen**

Show export buttons, then briefly show the repository test badge or terminal with passing tests. End on the product name.

## Recording rules

- Target duration: 2:25–2:45; never exceed 3:00.
- Use 1440×900 or 1920×1080.
- Increase browser zoom only if labels become difficult to read.
- Keep the cursor deliberate; avoid circling or random movement.
- Do not open source code for longer than five seconds.
- State the limitation clearly; it increases credibility rather than weakening the pitch.
